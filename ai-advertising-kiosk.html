AI-Based Smart Advertising Kiosk

Published: September 28, 2025 • 4 min read • Category: Computer Vision / IoT / Full-Stack

Overview

This project proposes a smart advertising kiosk built on a Raspberry Pi 5 (16GB). The kiosk detects people in front of the screen, estimates their age group and gender, and then selects the most relevant advertisement in real-time using rule-based decision logic.

Core Problem

Traditional digital signage shows the same ads to everyone. This project improves relevance by adapting content to the audience in front of the kiosk — while still being lightweight enough to run fully on-device.

How It Works

1) User Detection & Analysis

  • Camera captures frames continuously (about 10–15 FPS is enough).
  • Faces are detected and each face is classified into an age group (e.g., 12–17, 18–24, 25–34, 35–50, 50+).
  • Gender is estimated (male/female; can be extended to “unknown”).

2) Attention Tracking (Single User Only)

  • If exactly one user is detected, the system tracks attention using head pose / face landmarks.
  • If attention drops for a few seconds, the kiosk triggers a re-engagement response (avatar animation or short voice line).

3) Ad Selection & Decision Logic

  • If multiple users exist, the kiosk selects the dominant age group (most users).
  • If there’s a tie, it breaks the tie using a buying-power rule (e.g., older groups may win).
  • Gender is only used when it is reliable (single user or same gender within selected group).
  • The system queries an ads database (SQLite) and displays the best match.

AI Avatar & Voice Interaction

The kiosk includes a voice-enabled AI avatar that can wake up with a phrase like “Hey Adorix”. A 2D avatar is recommended for performance on Raspberry Pi, with 3D listed as future work.

Tech Stack (Planned)

  • Backend: Python (OpenCV), TFLite/ONNX runtime, Flask/FastAPI
  • Frontend UI: HTML/CSS/JavaScript (full-screen kiosk in Chromium)
  • Database: SQLite (ads, products, logs)
  • Optional Voice: Vosk (offline STT), eSpeak/PicoTTS (offline TTS)

Challenges & Ethics

  • Performance vs accuracy: lightweight models are needed on-device.
  • Lighting & angles: real-world kiosk environments affect detection.
  • Privacy: avoid saving face images; keep processing on-device where possible.
  • Voice reliability: background noise can reduce speech accuracy.

Future Enhancements

  • Upgrade from 2D to advanced 3D avatar (stronger hardware required).
  • Cloud dashboard for analytics (impressions, demographics, engagement).
  • Multi-language support and better accessibility.
  • More advanced ad selection using ML (optional).

This idea and architecture is based on a full project plan designed specifically to run on Raspberry Pi 5, including module breakdown, decision rules, database structure, and deployment approach.
